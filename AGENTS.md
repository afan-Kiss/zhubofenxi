# AGENTS.md — AI 协作规则

本项目是 **小红书直播经营 BI 看板**，供老板查看直播经营数据，**不是**财务对账、利润核算或人事提成系统。

技术栈：Electron / React / Vite / Tailwind（前端）+ Express / Prisma / SQLite（后端）。

---

## 项目定位

- **只做**：直播经营数据展示与分析（支付、订单、退款、品退、主播表现、客户画像）。
- **不做**：财务对账、平台结算、利润分析、提成、工资、扣款、责任归因。

---

## 主页面结构（不得随意增删）

当前主菜单仅保留四页：

1. **经营总览** — 经营大盘
2. **主播业绩** — 主播经营表现
3. **买家排行** — 全量客户画像（不随日期切换）
4. **系统设置** — 直播号、同步、数据维护（需管理密码）

**禁止**新增「订单明细」主菜单或独立订单列表页作为主入口。

---

以下方向 **不得新增或恢复为主功能**：

## 数据与 UI 硬性规则

1. **金额**：完整显示，例如 `¥10,079.90`；禁止 `¥1.01万` 等缩写。
2. **文案**：温和、经营向；禁用 TOP、排名、贡献、黑名单、责任、扣款、提成、工资等词（见 `docs/UI_COPY_RULES.md`）。
3. **多直播号**：首页大盘合并展示；仅在 Drawer、主播详情、买家详情等明细中展示「来源直播号」。
4. **买家排行**：全量历史画像缓存，**不**随经营总览日期切换。
5. **长周期指标**：今日 / 昨日 / 本周不强行展示签收率、品退率；月度 / 上月 / 自定义周期更适合展示。
6. **口径**：涉及金额、退款、品退、订单数时，必须写明统计口径与验收方法（见 `docs/DATA_METRICS_SPEC.md`）。

## AI 修改代码要求

### 工作方式

1. **先定位根因**，再改代码；禁止未验证假设就大面积重写。
2. **小范围快修优先**；不允许为了一个 bug 做无关重构。
3. **每次只解决当前任务**；不顺手改口径、布局、数据库。
4. **修改前先阅读**相关前后端代码与 `docs/` 规范。
5. **修改后必须运行** `npm run acceptance`（或至少 `npm run build` + 相关专项 accept 脚本）。

### 交付说明（每次改完必须写）

- 修改了哪些文件
- 根因是什么
- 验收步骤（含日期范围、指标口径）
- 影响范围（哪些页面 / API / 缓存）

### 数据口径变更

若改动涉及金额、退款、品退、订单数、买家标识：

- 必须同时检查 **前端展示** 与 **后端聚合** 是否一致
- 必须更新或对照 `docs/DATA_METRICS_SPEC.md`
- 必须说明验收数据（可参考黄金口径 2026-05-28）

# 验收清单

每次修改代码（尤其是金额、退款、品退、同步、买家排行）后，按本清单验收。自动化入口：

```bash
npm run acceptance
```

---

## A. 自动化检查（必跑）

| 命令 | 检查内容 |
|------|----------|
| `npm run check:business` | 治理文档存在且含核心原则 |
| `npm run check:copy` | 源码禁用词（相对 baseline 无新增） |
| `npm run check:finance` | 主路由/主菜单无财务/订单明细模块 |
| `npm run check:money` | 金额格式化无「万」缩写 |
| `npm run check:quality-return` | 服务端品退逻辑静态存在 |
| `npm run build` | TypeScript 编译与前端构建 |

---

## B. 构建与页面可达

- [ ] `npm run build` 通过
- [ ] 经营总览 `/` 可正常打开
- [ ] 主播业绩 `/anchors` 可正常打开
- [ ] 买家排行 `/buyers` 可正常打开
- [ ] 系统设置 `/settings` 可正常打开（管理密码）

---

## C. 产品结构（不得破坏）

- [ ] 主菜单仍为：经营总览、主播业绩、买家排行、系统设置
- [ ] **没有**新增财务 / 利润 / 提成 / 工资 / 对账 **主模块**
- [ ] **没有**订单明细 **主菜单**
- [ ] 金额展示 **无「万」缩写**
- [ ] UI **无** TOP、贡献、黑名单、责任、扣款等禁用词（新增文案）

---

## D. 品退相关（改动品退/售后时必查）

- [ ] 经营总览 — 品退单数 / 品退率（长周期范围）
- [ ] 主播业绩 — 品退指标
- [ ] 买家排行 — 品退重度客户
- [ ] Drawer — 品退明细列表
- [ ] 明细 — 售后原因展示正确
- [ ] 本月有真实品退数据时，主指标 **不为 0**

---

## E. 日期范围（改动 range / 率指标展示时必查）

逐一切换并目视/接口核对：

- [ ] 今日
- [ ] 昨日
- [ ] 本周
- [ ] 本月
- [ ] 上月
- [ ] 自定义（如 2026-05-28 ~ 2026-05-28）

确认：

- 短周期（今日/昨日/本周）不强行展示误导性签收率/品退率
- 长周期率指标与 Drawer 明细一致

---

## F. 金额口径（改动支付/退款时必查）

对照 `docs/DATA_METRICS_SPEC.md`：

- [ ] 支付金额按 **支付时间 + 已支付** 计入，退款 **不冲减**
- [ ] 退款金额 **单独** 统计
- [ ] 前后端同一 rangeKey 下数字一致
- [ ] 可选：黄金样例 2026-05-28 → 支付 10079.90 / 9 单 / 退款 29.90

---

## G. 买家排行（改动买家画像时必查）

- [ ] 不随经营总览日期切换
- [ ] 重建买家排行 **不触发** 30 分钟经营同步
- [ ] 经营同步中：有缓存仍展示排行；无缓存显示买家画像空态/重建态
- [ ] **不**展示经营同步订单列表页码进度

---
